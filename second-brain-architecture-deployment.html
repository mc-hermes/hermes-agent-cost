Second Brain — How GitHub Pages Fits

One brain, two surfaces. VPS is the engine. GitHub Pages is the billboard.

VPS :8080 — The Engine

  • • Runs gbrain (queries, ingest, embed)
  • • Serves LIVE dashboard with fresh JSON
  • • /api/query endpoint for semantic search
  • • Cron polls Gmail, processes emails
  • • Source of truth for ALL data

GitHub Pages — The Billboard

  • • Static snapshot pushed every 2 min
  • • Public URL — anyone can view
  • • Landing page, articles, architecture
  • • Dashboard with stale(ish) JSON
  • • No query capability (no backend)

The Demo Strategy

  • • Send prospects to GitHub Pages first
  • • Landing page → business case → PRD
  • • For live demos: VPS :8080 dashboard
  • • Query box only works on VPS
  • • Both surfaces show same data shape
