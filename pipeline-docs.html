Email to Brain Pipeline

A system that lets you forward any email to a special address, review it on a dashboard, and decide whether to add it to your permanent knowledge base — before it ever touches the database.

How +aliases work: Gmail delivers info+anything@malohacoast.com to the same info@ inbox — for free. The +anything suffix tells the system which workspace the email belongs to. No new accounts, no extra cost.

The Four Stages

1. Forward an Email

Forward any email to one of the addresses above. That's it. The system picks it up automatically.

2. Pipeline Processes It

A background process runs twice daily (9 AM and 9 PM) plus whenever new mail arrives (max once every 3 hours). It:

  • Reads the email and classifies it (newsletter, meeting notes, article, etc.)
  • Creates a clean markdown file with the content
  • Scans for entity names (people, companies, concepts) mentioned in the email
  • Checks which entities already exist in the knowledge base
  • Saves everything as a pending item — NOT yet in the brain

3. Review on the Dashboard

Open your dashboard and click the Inbox tab. Each item shows:

  • 📧 Subject and sender
  • 🏷️ Type classification
  • 📄 The page that will be created
  • 🆕 New entities (people/companies) that will be created
  • 🔗 Links to existing pages already in the brain

You see exactly what will happen before you click anything.

4. Approve or Reject

Approve: The content is written into the knowledge base, links are created, and the dashboard updates. The page is now searchable and linkable.

Reject: The content is moved to a rejected folder. It never enters the brain. It cannot be searched, linked, or found by accident.

The Knowledge Base

Under the hood, the brain is a system called gbrain — think of it as a personal Wikipedia:

  • Pages — each approved item becomes a page
  • Links — [[Ventris]] in any page auto-links to the Ventris page
  • Types — pages are tagged as person, company, meeting, newsletter, etc.
  • Search — ask natural questions like "What do we know about the Cradle grant?"

The Database (for technical readers)

The brain uses PGLite — PostgreSQL compiled to WebAssembly, stored as a single file. No database server, no Docker, no connection strings. Each approved page creates rows in the pages table, chunks table, and vector embeddings for semantic search.

Rejected items never touch PGLite. The markdown file is simply moved to a rejected folder. This keeps the database clean and fast.

New Workspace Setup

Adding a new client or project workspace:

  1. Give them the address: info+newname@malohacoast.com
  2. Add one line to the pipeline configuration
  3. Copy the dashboard template, change the workspace name

That's it. No new accounts, no new servers, no extra cost.
